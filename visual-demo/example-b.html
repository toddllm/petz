Example B: Database Load Scenario

Loading from SQLite Storage

When a player moves to a new area or returns after blocks have been unloaded from cache, Luanti must load the MapBlock from the SQLite database. This involves decompression and remapping of node IDs.

Starting position: Player at (-2044, 2029, -2043) in underground mine
Block position: (-128, 126, -128)
Cache status: MISS - must load from database

1

Cache Miss

After calculating block position (-128, 126, -128), the cache lookup fails. Time to load from database!

// src/map.cpp MapBlock *Map::getBlockNoCreateNoEx(v3s16 p) { MapBlock *block = m_blocks_cache.lookup(p); if (block) return block; // Cache miss - need to load from database block = loadBlock(p); if (block) m_blocks_cache.insert(p, block); // Add to cache for next time return block; }
2

Encode Position for Database

Convert the 3D block position into a 64-bit integer key for SQLite storage.

// src/database/database-sqlite3.cpp static s64 getBlockAsInteger(const v3s16 &pos) { return (u64)(pos.Z + 2048) << 24 | (u64)(pos.Y + 2048) << 12 | (u64)(pos.X + 2048); } // For position (-128, 126, -128): // key = ((-128 + 2048) << 24) | ((126 + 2048) << 12) | (-128 + 2048) // key = (1920 << 24) | (2174 << 12) | 1920 // key = 32212320915328
3

SQLite Query

Execute the database query to retrieve the compressed block data.

// SQL query executed: SELECT data FROM blocks WHERE pos = 32212320915328; // Returns BLOB: 1D 28 B5 2F FD 00 58 DD 02 00... // Size: 621 bytes (ZSTD compressed)

Database Response:

Version byte: 0x1D (29)
Compressed data: 620 bytes of ZSTD data
4

ZSTD Decompression

Decompress the block data from ~600 bytes to ~16KB of raw data.

// src/serialization.cpp void decompressZstd(std::istream &is, std::ostream &os) { ZSTD_DStream* stream = ZSTD_createDStream(); ZSTD_initDStream(stream); // Decompress in chunks while (input_available) { size_t ret = ZSTD_decompressStream(stream, &output, &input); os.write(output_buffer, output.pos); } } // Result: ~16KB of decompressed block data
5

Parse NameIdMapping

Read the block's unique node type mapping. This block uses 7 different node types.

// src/nameidmapping.cpp void NameIdMapping::deSerialize(std::istream &is) { u16 count = readU16(is); // count = 7 for (u16 i = 0; i < count; i++) { u16 id = readU16(is); std::string name = deSerializeString16(is); m_id_to_name[id] = name; m_name_to_id[name] = id; } }

This Block's Mapping:

0 ‚Üí "default:stone"
1 ‚Üí "default:stone_with_iron"
2 ‚Üí "default:stone_with_coal"
3 ‚Üí "default:stone_with_copper"
4 ‚Üí "default:gravel"
5 ‚Üí "default:silver_sand"
6 ‚Üí "default:stone_with_tin"
6

Read Node Array

Read all 4096 nodes (16√ó16√ó16) with their content IDs and parameters.

// src/mapnode.cpp void MapNode::deSerializeBulk(std::istream &is, MapNode *nodes, u32 nodecount) { for (u32 i = 0; i < 4096; i++) { nodes[i].param0 = readU16(is); // content_id (local) nodes[i].param1 = readU8(is); // light values nodes[i].param2 = readU8(is); // rotation/param2 } } // Total: 4096 nodes √ó 4 bytes = 16,384 bytes
7

Remap IDs: Local to Global

Convert block-local IDs to global content IDs using the node definition manager.

// src/mapblock.cpp static void correctBlockNodeIds(NameIdMapping *nimap, MapNode *nodes, IGameDef *gamedef) { const NodeDefManager *nodedef = gamedef->ndef(); for (u32 i = 0; i < 4096; i++) { content_t local_id = nodes[i].param0; std::string name; // Get name from block's mapping if (nimap->getName(local_id, name)) { // Get global ID from node registry content_t global_id; nodedef->getId(name, global_id); nodes[i].param0 = global_id; } } } // Example: local ID 2 ‚Üí "default:stone_with_coal" ‚Üí global ID 97
8

Return Requested Node

Finally, calculate the array index and return the specific node data.

// Calculate position within block v3s16 relpos = world_pos - blockpos * 16; // Get node at calculated index u32 index = relpos.Z * 256 + relpos.Y * 16 + relpos.X; MapNode node = block->data[index]; // node.param0 = 97 (global ID for "default:stone_with_coal") // Ready for game to use!
